Swift & SwiftUI Cheatsheet

A quick reference for Swift & SwiftUI, covering variables, control flow, collections, views, and state management.

Swift Cheatsheet

Comments

Comments are notes for developers and are ignored by the compiler. Use // for single-line comments and /* ... */ for multi-line comments.

// This is a single-line comment.
// It's useful for short explanations.

/*
 This is a multi-line comment.
 It can span several lines and is often
 used for longer descriptions or for temporarily
 disabling a block of code.
*/
let a = 5 // Comments can also be at the end of a line.

Output

Comments do not produce any visible output in your app but are crucial for code readability.

Variable & Constant

Use `var` for data that can change (like a score) and `let` for data that will never change (like a birthday).

var score = 0  // Variable (mutable)
let pi = 3.14  // Constant (immutable)

Basic Data Types

Swift needs to know the "type" of data you are storing. Common types include text (`String`), whole numbers (`Int`), decimals (`Double`), true/false (`Bool`), and collections like arrays (`[String]`, `[Int]`).

Note: Swift has powerful type inference, so you often don't need to write the type explicitly. var name = "Alex" works just fine!

// MODIFIED: Using var for all examples
var name: String = "Alex"
var age: Int = 16
var height: Double = 1.75
var isStudent: Bool = true

// Arrays
var teamNames: [String] = ["Warriors", "Lakers", "Bulls"]
var scores: [Int] = [98, 102, 85]

Random Numbers & Elements

Swift makes it easy to get random values, whether from a numerical range or from a collection like an array.

// Random element from an array
let fruits = ["Apple", "Banana", "Cherry"]
if let randomFruit = fruits.randomElement() {
    print("Random fruit: \(randomFruit)")
}

Output

If/Else Statements

Use if, else if, and else to make decisions in your code based on certain conditions.

let temperature = 25
if temperature > 30 {
    print("It's a hot day!")
} else if temperature < 15 {
    print("It's a cold day.")
} else {
    print("The weather is mild.")
}

Output

The weather is mild.

For Loops

for-in loops allow you to repeat a block of code for each item in a sequence, like an array or a range of numbers.

// Looping through an array
let names = ["Anna", "Alex", "Brian"]
for name in names {
    print("Hello, \(name)!")
}

// Looping through a range of numbers
for number in 1...3 {
    print("Number is \(number)")
}

Output

Hello, Anna!
Hello, Alex!
Hello, Brian!
Number is 1
Number is 2
Number is 3

SwiftUI Cheatsheet

Layout

VStack

Use `VStack` to arrange views in a vertical line, one on top of the other, like stacking blocks.

VStack(spacing: 10) {
    Text("Item 1")
    Text("Item 2")
}

Output

Item 1
Item 2

HStack

Use `HStack` to arrange views in a horizontal line, side-by-side, like books on a shelf.

HStack(spacing: 10) {
    Text("Left")
    Text("Right")
}

Output

Left
Right

Spacer

A `Spacer` is a flexible, invisible view that pushes other views as far apart as possible. It's great for aligning things to the edges.

HStack {
    Text("Start")
    Spacer()
    Text("End")
}

Output

Start
End

ZStack

ZStack overlays views on top of each other, like stacking transparent sheets. The first view in the ZStack is at the bottom, and subsequent views are layered on top.

ZStack {
    Color.yellow // Bottom layer
    Text("Centered Text") // Top layer
}
.frame(width: 150, height: 100)

Output

Centered Text

Combining Layouts

VStack, HStack, ZStack, and Spacer can be nested and combined to create complex user interfaces. This example shows a card-like structure.

ZStack(alignment: .topLeading) {
    // Background for the card
    Color.gray.opacity(0.1)
        .frame(width: 250, height: 150)
        .cornerRadius(10)

    VStack(alignment: .leading, spacing: 8) {
        HStack {
            Image(systemName: "star.fill")
                .foregroundColor(.yellow)
            Text("Featured Item")
                .font(.headline)
            Spacer() // Pushes content left
        }
        Text("This item uses ZStack, VStack, HStack, and Spacer.")
            .font(.caption)
    }
    .padding() // Padding inside the VStack
}
.frame(width: 250, height: 150) // Frame for ZStack

Output

Featured Item

This item uses ZStack, VStack, HStack, and Spacer.

Views & Modifiers

Text View: Displaying Text

The Text view is the most fundamental way to display read-only text on the screen.

Text("Hello, SwiftUI!")

Output

Hello, SwiftUI!

Image View: Using SF Symbols

Use Image(systemName:) to display icons from Apple's extensive SF Symbols library. Here are a few examples. These symbols are vector-based and adapt well to different sizes and styles.

// Displaying various SF Symbols
VStack(spacing: 15) {
    Image(systemName: "star.fill")
    Image(systemName: "heart.circle.fill")
    Image(systemName: "trash.slash.fill")
    Image(systemName: "folder.badge.plus")
}
.font(.largeTitle) // Apply a consistent size
.foregroundColor(.blue) // Apply a consistent color

Output

Image View: Using Project Images

To display an image from your project's Asset Catalog (e.g., an image named "RMIT-banner"), use Image("your-image-name").

// Assuming "RMIT-banner" is in your Asset Catalog
Image("RMIT-banner")

Output

Common Modifiers: Basic Styling

Modifiers are chained to views to change their appearance or behavior. Common ones include .font(), .foregroundColor(), .padding(), and .background().

// For Text
Text("Styled Text")
    .font(.headline) // Changes text style
    .foregroundColor(.purple) // Changes text color
    .padding() // Adds space around the text
    .background(Color.yellow.opacity(0.3)) // Sets background

// For Image (SF Symbol)
Image(systemName: "mic.circle.fill")
    .font(.largeTitle) // Affects SF Symbol size
    .foregroundColor(.red) // Changes symbol color

Output

Styled Text

Image Modifiers: Sizing & Resizing

Control image dimensions with modifiers. .resizable() is crucial for project images to allow them to change size. .scaledToFit() fits the image within its frame while preserving aspect ratio. .scaledToFill() fills the frame, potentially cropping. .frame() sets specific dimensions.

// Fit within a frame
Image("RMIT-banner")
    .resizable()
    .scaledToFit()
    .frame(height: 100) // Width adjusts

// Fill a frame (may crop)
Image("RMIT-banner")
    .resizable()
    .scaledToFill()
    .frame(width: 250, height: 100)
    .clipped() // Good with scaledToFill

Output

Image Modifiers: Shaping & Effects

Style images with shapes, borders, and shadows. .clipShape() (e.g., Circle(), RoundedRectangle()) masks the image. .overlay() can add borders. .shadow() adds depth.

Image(systemName: "camera.fill")
    .font(.system(size: 50)) // Control SF Symbol size
    .padding(20) // Add padding before background/clip
    .background(Color.blue.opacity(0.2))
    .clipShape(Circle())
    .overlay(
        Circle().stroke(Color.blue, lineWidth: 2)
    )
    .shadow(color: .gray, radius: 3, x: 2, y: 2)

Output

Interactivity & Animation

State & Actions

Use `@State` to store a value that can change. A `Button` can change that value, and SwiftUI will automatically update the view.

@State private var tapCount = 0

VStack(spacing: 15) {
    Text("Taps: \(tapCount)")
    Button("Tap Me") {
        tapCount += 1
    }
}

Interactive Output

Taps: 0
Tap Me

Implicit Animation

The `.animation()` modifier tells SwiftUI to make any changes to this view smooth instead of instant. Tapping the circle will now animate its size change.

@State private var isScaled = false

Circle()
    .frame(width: 50, height: 50)
    .scaleEffect(isScaled ? 2 : 1)
    .onTapGesture { isScaled.toggle() }
    .animation(.easeInOut, value: isScaled)

Interactive Output

Navigation

NavigationView & NavigationLink

`NavigationView` creates a navigation bar. `NavigationLink` is a special button that pushes a new view onto the screen.

NavigationView {
    VStack {
        NavigationLink(destination: Text("Detail View")) {
            Text("Go to Detail")
        }
    }
    .navigationTitle("Master View")
}

Output: Master → Detail View

Master View
Go to Detail
→
< Master Detail View
Detail Content
